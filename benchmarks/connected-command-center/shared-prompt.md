# Shared Prompt

You are given a minimal Flutter application and deterministic fixture data for three domains: project delivery, monthly finances, and an upcoming trip.

Design and implement a production-quality command center that helps a maintainer understand what needs attention now.

Requirements:

- use one Flutter codebase for compact mobile and expanded tablet/desktop constraints;
- preserve all supplied models and callbacks;
- represent populated, loading, empty, and recoverable error states;
- keep unaffected domain content useful when one source fails;
- support keyboard navigation, meaningful semantics, enlarged text, and reduced motion;
- add proportionate widget tests;
- render deterministic compact, expanded, and error screenshots;
- document important design and adaptive decisions.

Do not add a backend, network service, authentication flow, or unrelated package.
