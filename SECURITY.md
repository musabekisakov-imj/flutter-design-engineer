# Security Policy

Report vulnerabilities privately through GitHub Security Advisories rather than public issues. Include affected files, impact, reproduction steps, and a suggested mitigation when possible.

The project contains instructions and local installer scripts. Review changes before installing. The installer performs local file copies only, refuses overwrite by default, and does not download or execute remote content.

Security fixes are prioritized for the latest release. Maintainers will acknowledge credible reports, assess impact, and coordinate disclosure.
