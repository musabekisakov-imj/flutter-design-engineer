# GitHub social preview

## Goal

Give repository links a recognizable Flutter Design Engineer identity without changing the owner's avatar across every GitHub repository.

## Approved design

- Produce a 1280 × 640 social card in the existing dark navy, lime, cyan, and white system.
- Place the exact website mark and product name at the upper left.
- Use the message `A senior Flutter design team, inside your agent.` with the supporting line `Seven connected skills · one verified workflow`.
- Show the real Product Studio workspace and release golden-test screens on the right.
- Include a restrained install-command strip as developer-tool context.
- Avoid fake metrics, gradients unrelated to the current brand, or generated application screens.
- Reuse the card in repository README and website Open Graph metadata.
- Upload the rasterized 1280 × 640 PNG as GitHub's repository Social Preview.

## Verification

Confirm exact dimensions, readable crop at small sizes, real asset provenance, valid README and Open Graph paths, repository checks, and the saved Social Preview state in GitHub settings.
