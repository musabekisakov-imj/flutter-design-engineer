## Maintainer statement

I maintain Flutter Design Engineer, an open-source modular agent-skill system for building adaptive, accessible, and visually verified Flutter interfaces. It addresses an underserved gap in the coding-agent ecosystem: most design skills focus on web frontends, while Flutter teams lack a rigorous product-to-render workflow.

The project provides seven self-contained skills, progressive design-engineering references, behavioral evaluation rubrics, deterministic repository validation, a safe installer, compile-verified Flutter starter assets, and a real adaptive Flutter fixture with committed golden screenshots. Its workflow prevents agents from jumping directly from vague requests to widget code; it requires product context, explicit visual direction, state coverage, accessibility checks, rendered evidence, and refinement.

I plan to use Claude Code to maintain the skills, evaluate behavior across real Flutter codebases, expand platform and localization coverage, review community contributions, and improve visual-regression workflows.

This is a new project and I am applying through the program's “quietly depended on / does not quite fit” consideration. I am not claiming that it currently meets the published dependent-repository, download, contributor, pull-request, or OpenSSF thresholds. The public release, CI, evals, demo, roadmap, and contribution infrastructure are available in the repository.
