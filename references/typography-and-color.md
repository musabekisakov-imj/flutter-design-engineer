## Typography

Define semantic roles for display, title, body, label, caption, and numeric data. Use the smallest type scale that creates clear hierarchy. Test real translations, long names, prices, dates, and accessibility scaling.

Prefer weight, size, leading, width, and placement before adding color. Use tabular figures for changing aligned numbers where the font supports them. Avoid truncating critical information solely to preserve composition.

## Color roles

Start with semantic roles: canvas, surface, raised surface, primary content, secondary content, border, accent, success, warning, error, selection, focus, and disabled. Map roles to `ColorScheme` and narrow theme extensions.

Use accent sparingly enough that it retains meaning. Communicate status through iconography, text, or structure in addition to color.

## Themes and contrast

Design dark mode as its own luminance system; do not mechanically invert. Control adjacent surface separation, glare, saturated accents, overlays, and elevation. Verify contrast for text, icons, focus, charts, and disabled controls using the project's target accessibility standard.
