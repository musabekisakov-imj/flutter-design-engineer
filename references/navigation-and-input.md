# Navigation and Input

## Contents

- Navigation
- Forms
- Selection and destructive actions

## Navigation

Match navigation depth and destination count to platform conventions. Preserve back behavior, deep links, selected state, scroll position, and restoration. Avoid hiding frequent destinations behind ambiguous icons.

## Forms

Use persistent labels where recall matters. Connect errors to fields, explain recovery, preserve user input, select appropriate keyboards/autofill, and make submission state explicit. Validate at the right time: early enough to help, not so early that untouched fields appear broken.

## Selection and destructive actions

Make selected state visible without color alone. Put destructive actions away from routine primary actions, state consequences precisely, and offer undo when reversal is safe and simpler than confirmation.
